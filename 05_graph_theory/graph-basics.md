# Graph Basics

## Definition (EN)
Graph is a set of nodes connected by edges.

## Определение (RU)
Граф — это набор вершин, соединённых рёбрами.

Вершина — это точка, которую мы рассматриваем как отдельный объект графа.
В нашей модели вершинами будут:
ConnectivityNode
ConductingEquipment

Ребро — это связь между двумя вершинами.
В нашей задаче ребро означает:
есть допустимое соединение между узлом связности и оборудованием
Логически:
ConnectivityNode <-> ConductingEquipment

Путь — это последовательность вершин, по которой можно пройти от одной вершины к другой.
Пример логики:
источник
узел
выключатель
другой узел
линия
следующий узел
Если такой путь существует и проход через все аппараты разрешён, участок считается достижимым.

Достижимость — это главный смысл всей задачи.
Если от источника есть путь до вершины, значит:
вершина достижима
вершина под напряжением
Если пути нет:
вершина недостижима
вершина не под напряжением

Связная компонента — это группа вершин, между которыми существуют пути.
Для топологии это полезно так:
если источник попал в компоненту, то можно проверить, какие элементы этой компоненты достижимы
если участок вообще в другой компоненте, напряжение туда не попадёт

Цикл — это когда по связям можно вернуться в уже пройденную вершину.
В подстанции это возможно:
через секционные связи
обходные связи
кольцевые участки
Поэтому нужен признак:
вершина уже посещена
повторно не обходить

Что такое множество посещённых вершин
Это список или набор вершин, которые уже были обработаны при обходе.
Он нужен, чтобы:
не зациклиться
не обходить одно и то же много раз
сделать расчёт устойчивым
