# Coloring Rules

## Principle (EN)
Coloring of elements is defined by project requirements, standards, or customer specifications.

## Принцип (RU)
Раскраска элементов определяется требованиями проекта, стандартами или спецификацией заказчика.

---

## Typical Approach (EN)
- Energized state → defined by project (e.g. green / red / other)
- De-energized state → defined by project (e.g. gray / white / other)

## Типовой подход (RU)
- Под напряжением → определяется проектом (например: зелёный / красный / другой)
- Без напряжения → определяется проектом (например: серый / белый / другой)

---

## Important Notes
- Coloring must be configurable
- Logic must be separated from visualization
- SCADA should not hardcode colors

## Важные замечания
- Цвета должны быть настраиваемыми
- Логика должна быть отделена от визуализации
- Цвета не должны быть жёстко зашиты в коде
